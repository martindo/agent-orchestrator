# Audit Task List

Source: full-codebase audit, 2026-07-22 (six parallel subsystem audits). Check items off as completed.

**Verdict:** This is a substantially real platform ‚Äî the orchestration loop makes real LLM calls, all 11 connectors make real API calls, MCP uses the real SDK, Studio is wired end-to-end. But the "Phase 34 Complete / All Features Implemented / 1451 tests passing / zero known issues" claim is false in specific, load-bearing ways: **no auth anywhere, a PostgreSQL backend that doesn't exist, governance running on a constant, an audit trail that isn't tamper-evident, and latent bugs hidden behind swallowed exceptions.**

> **‚úÖ Test baseline (measured 2026-07-22):** the suite now *runs* (install PyPI via `pip install --trusted-host pypi.org --trusted-host files.pythonhosted.org -e ".[dev,llm,mcp]"` ‚Äî plain install hit a corporate-CA SSL error). **The "1451 passing / zero known issues" claim was false.** As shipped, the `tests/` suite was **1342 passing, 14 failing, +1 flaky** ‚Äî and it couldn't even be collected without the optional `llm`/`mcp` extras (a `cannot import name 'AnthropicProvider'` collection error breaks *all* tests when those SDKs are absent). The 14 failures were **stale tests**: connector providers had grown operations (Slack gained `list_channels`/`upload_file`/`send_notification`; repository providers gained *write* ops `create_pull_request`/`create_issue`/`add_review_comment`; ticketing gained `transition_ticket`/`get_sprint_issues`) and the MCP manager now raises the more-specific `MCPConfigurationError` ‚Äî none of the assertions were updated because **there is no CI** (Tier 5.1) running them. After the fixes below the suite is **1357 passing, 0 failing**, deterministic across repeat runs. Studio's separate suite was not run here.

Effort tags: **S** = under a day, **M** = 1‚Äì3 days, **L** = a week+.

## Concurrency & dependency map (remaining open tasks)

No open task has a hard **logical** prerequisite on another ‚Äî they are independent fixes. The only constraint is **file collisions**: tasks that edit the same file must be serialized or bundled into one PR; tasks in different groups are safe to run in parallel. Rule of thumb: **pick at most one task per Group at a time**; anything with Group `‚Äî` is a standalone file and parallel-safe with everything.

## Tier 3 ‚Äî Make the core promises actually true

- [x] **3.1 (M)** **Confidence-based governance now runs on real signal.** ‚úÖ 2026-07-22 ‚Äî `_build_user_prompt` now asks the agent to end with a `CONFIDENCE: <0..1>` marker; `output_parser.parse_confidence` extracts it and `agent_executor._attach_confidence` injects it into the output dict (leaving an explicit value untouched, adding nothing when absent ‚Äî no fabrication). So `extract_confidence`/`aggregate_confidence` see real values instead of the flat 0.5, and the governor's thresholds actually differentiate. End-to-end test proves a real `0.05 ‚Üí ABORT`, `0.3 ‚Üí QUEUE_FOR_REVIEW`, `0.95 ‚Üí ALLOW` (previously every output was a fixed `ALLOW_WITH_WARNING`). 19 tests. Full suite ‚Üí 1441 pass.
- [x] **3.2 (M)** **Audit trail is now tamper-evident.** ‚úÖ 2026-07-22 ‚Äî `_compute_hash` hashes the **full** record (all fields incl. the `data` payload) with the **full** SHA-256 digest (was 7 fields truncated to 64 bits, `data` unprotected); `verify_chain` now **recomputes** every record's hash to detect content edits *and* walks all rotated segments checking `prev_hash` linkage (was link-pointers only, current file only); rotation no longer resets `_last_hash`, so the chain spans segments (dropping a whole segment is now detected), and rotated filenames use microsecond precision (unique + chronologically sortable). 11 new tamper-evidence tests (payload edit, field edit, reorder, delete, cross-rotation tamper, dropped segment). Makes the "tamper-evident audit trail" claim true (also clears part of 4.7).
- [x] **3.3 (S)** `decision_ledger` hash now covers every field. ‚úÖ 2026-07-22 ‚Äî `_compute_record_hash` enumerated only 18 fields, silently leaving `tool_calls`, `warnings`, `review_notes`, `duration_seconds`, `metadata` outside the chain (editable despite the "immutable" claim, and the docstring lied about "any field"). Refactored to a shared `_hash_record_dict` that hashes the **full** record dict (every field except `record_hash`); `_write_record` and `verify_chain` now use the same canonical representation, so verify recomputes over all fields instead of a hand-picked subset. 16 tests (per-field tamper detection incl. the 5 previously-unprotected fields). Full suite ‚Üí 1422 pass.
- [x] **3.4 (M)** **Restart durability ‚Äî incomplete work recovered.** ‚úÖ 2026-07-22 ‚Äî `engine._recover_incomplete_work()` (called from `start()`) re-enqueues non-terminal items from `work_item_store.get_incomplete()`; items that were mid-pipeline have their position restored (`enter_pipeline` + `reset_to_phase`), and the processing loop now **resumes** an already-present entry instead of failing on it. Queued/in-flight work is no longer orphaned on restart. Verified end-to-end (persist an orphan ‚Üí fresh engine drives it to completion).
